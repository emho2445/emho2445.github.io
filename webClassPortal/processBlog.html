Class Portal - Milestone 1

September 6, 2023

I am going to host my Class Portal on my portfolio website. I started this website about a year ago and my goal is to have it be something I can showcase that shows both my web development abilities and the other technical projects that I have worked on. As shown on the first page of the wireframe below, this portal is going to be located under “Class Blogs” since I have several classes that need a blog-like portfolio. Within the class portal I plan to have four different pages, Main, Projects, Assignments and Process Blog. Each of these pages can be seen on pages 2-9 wireframe.

On each wireframe page I created boxes that outline the general location of the information. I do not plan to have each item in a box with borders but the boxes help me visualize the space on the page which content will be confined to. I am also going to have to learn about how to create the drop-down menu style since that is not currently implemented in this website.

The main page of the Class Portal will have three buttons at the top of it that link out to the other pages where the assignments, projects and blog posts are going to be turned in. This page is also intended to give context to a potential employer or someone without knowledge of the class.

My portfolio has an overall minimalist appearance to it that I plan to keep. I want to keep readers focused on the content without a lot of distractions on the page. Something that I want to improve as far as layout is image sizing. I always feel that my images are either too big and break up the text too much or are too small to see all of the details.

Project 1 - Milestone 1: Proposal

October 12, 2023

Project Overview:

For Project 1 I would like to create a website to help my job with presenting data sets. Right now we are using screenshots from code environments to display data. I would like to use the D3.js library to figure out how to represent that data in an interactive way. The audience for this project are engineers on my project that have to handle data taken through testing.

Use of Javascript:

This project is going to heavily rely on JavaScript as all of the data visualizations will be done through JavaScript. By using JavaScript it will make the graphs and 3D data models interactive instead of stationary images. I would also like to use Javascript to enhance the aesthetics of the website. I want to include hover options and other event listeners to give users input as they go throughout the website.

Use of Javascript Library:

After doing some research I would like to use D3.js to represent the data. They have features for interactive data sets, 3D graphing, geo-mapping and more. I think that it would be really cool to use the interactive visualizations to explore the different tests that we do. I also really want to create a 3D map that shows all of our global teams (we have 220 teams from 30 different countries) and it would be so amazing to have a visualization of those teams that isn't just a google map with markers that would really help our marketing.

Resources:
  • Various spreadsheets internal to work that has all our testing data
    • Antenna signal strength
    • Micrometeorite impact testing
    • Housing and Deployment Prototype distance testing
  • Locations of global teams which are found in our participant information spreadsheet
Responsive Design:

To make this responsive I would like to use media queries that will change the layout of the page for different sized screens. I will also explore the extent of D3.js to see what features they have to make their visualizations work on different devices.

Goals

Prototyping/proof of concept:
  • I want to have the base of the website built by October 19
  • I want to have all of my data in the correct format for the libraries by October 19
  • Talk to my team about the best way to represent all of the data
Minimal Viable Product
  • I want to have at least static 2D and 3D models
  • I want to have an interactive map of all of the teams from across the world
Stretch Goals
  • Create something that can take in data through the website
  • Explore different aesthetic minimalist design using my project's branding guide
  • Making all of the data interactive in some aspect
