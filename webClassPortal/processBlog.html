Project 1 - Milestone 1: Proposal

October 12, 2023

Project Overview:

For Project 1 I would like to create a website to help my job with presenting data sets. Right now we are using screenshots from code environments to display data. I would like to use the D3.js library to figure out how to represent that data in an interactive way. The audience for this project are engineers on my project that have to handle data taken through testing.

Use of Javascript:

This project is going to heavily rely on JavaScript as all of the data visualizations will be done through JavaScript. By using JavaScript it will make the graphs and 3D data models interactive instead of stationary images. I would also like to use Javascript to enhance the aesthetics of the website. I want to include hover options and other event listeners to give users input as they go throughout the website.

Use of Javascript Library:

After doing some research I would like to use D3.js to represent the data. They have features for interactive data sets, 3D graphing, geo-mapping and more. I think that it would be really cool to use the interactive visualizations to explore the different tests that we do. I also really want to create a 3D map that shows all of our global teams (we have 220 teams from 30 different countries) and it would be so amazing to have a visualization of those teams that isn't just a google map with markers that would really help our marketing.

Resources:

    • Antenna signal strength
    • Micrometeorite impact testing

  • Locations of global teams which are found in our participant information spreadsheet
Responsive Design:

To make this responsive I would like to use media queries that will change the layout of the page for different sized screens. I will also explore the extent of D3.js to see what features they have to make their visualizations work on different devices.

Project 1 - Milestone 4: Reflection

November 9, 2023

For project one I definitely ran into more challenges than I thought I would. I started with a fairly ambitious project of creating interactive 2D and 3D data as well as maps with information. I based this off of the examples that I saw on the D3.js website. However when I started coding the project I ran into issues with the data format and library capabilities.

My process was to start with the 2D data to get a grasp on the library and then move on to the harder 3D and map visualizations. Once I got my graphs in place I started working on interactive JavaScript. Then I created all of the information on the website that gives context to the website.

When I started with the 2D graphs, first I created a basic empty graph with two axes and this was fairly simple. I didn't run into any problems with creating the canvas for the graphs. Once I tried to start reading in my data points is when I started running into issues. The data that I had was a CSV file with headers but I didn't realize that all my data had to be objects with properties that were the data points. So I figured out a way to read in my data and then turn it into objects. Once that data was in the correct format making the line graph was very simple.

When I went to start looking at the 3D data and the map I realized that D3.js does not support those features on its own. All of the examples that I was looking at when I created my proposal had other imported libraries and data sets that were beyond the scope of what I was going to be able to learn in the given time. This was a huge barrier to my project. I decided that creating these data visualizations was not going to be possible.

By the end of this project I do feel much more confident using the D3.js library. I was able to figure out different types of scales for the axes, plots and line graphs. I also learned how to read their documentation. I think that this is a library that I would use again and I enjoyed working with it. In the future I might want to try some of the other libraries that people were using alongside it to do some of the more complicated data visualizations.

Project 2 - Milestone 1

November 16, 2023

Group Members:

Kaya Walker, Emma Hopson, Sydney Calcagno

Overall Project Description:

Our single page application (SPA) is to be a fun trivia game that gives the player different trivia questions within different categories. The inspiration for this SPA is the game Trivia Crack that became extremely popular around 2014. In Trivia Crack players would spin the wheel to get the question category, if they got the question correct they would get to spin the wheel for a new question. The goal was to get a character, which was a symbol of category mastery, before the other person. Our website will draw similar features from this game. We intend to have different question categories and present the user with different types of trivia questions. Our website will be different in key ways as ours is going to be single player and track overall success/progress instead of collecting icons.

Our audience will most likely be ages 16-35 as we intend to find an API that has cultural trivia meant for young adults and teens.

Sources:

For this project we are going to use a trivia API to generate all of the questions. The API that we chose is Open Trivia Database. This API is free and does not require an API key or account. This API also allows us to pull more than just the question and answer, it has other false answer options, a difficulty level and a category. This will allow us to potentially track the “expert” level of the player as well.

Link to Open Trivia Database: https://opentdb.com/api_config.php

Assignment 6: Plan a React To-Do List

November 16, 2023

For step one I created the mockup drawn above in dark navy blue. For step one I broke that mockup into the anticipated components. The green box outlines the overall application which is the searchable and filterable to-do list. Next the pink box outlines the search/filter bar. The orange outlines the add task component. The light purple outlines the overall task list. Each task is outlined in red.

The search/filter component will dictate the order in which the to-dos are displayed in the task list. The add component will also affect the task list as it can add new items to the task list.

Next I moved to step three to find my state. To find my minimal state I walked through the questions in the reading to determine what the state variables are. I determined that the state variables are the search text box and the filter checkboxes. These components change over time and are not able to be computed from other things. They also affect other aspects of the application.

The add button and filtered/unfiltered task list are not state. The add button itself does not change over time. The task list might change overtime but it is computed from other properties (props) that are passed down to it.

For step four I needed to determine where my state would live. Given the information above, I believe that the state should sit in the search/filter component that is outlined in pink. This would be able to pass state down to the task list items with the larger task list.
